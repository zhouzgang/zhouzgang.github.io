大致按照这个结构来，在结合一些源码看看具体是如何实现的，体会一下用来哪些设计模式来解决场景问题的。其实官网文档写的非常清晰了，学一个东西先过一篇官网文档是一个强制要养成的习惯。

MyBatis：ORM 框架，相对于 Hibernate 全自动 ORM 框架来说，MyBatis 是半自动的。

先看架构图：(拿来主义一张架构图，已征求原博主)

架构解析：

  1. 接口层： 和数据库交互的方式
    交互方式有两种：
     a.使用传统的MyBatis提供的API；
     b. 使用Mapper接口
  2. 数据处理层
    a. 通过传入参数构建动态SQL语句；
b. SQL语句的执行以及封装查询结果集成List<E>
  3. 框架支撑层
    a. SQL语句的配置方式
b. 缓存机制
c. 连接池管理机制
d. 事务管理机制
  4. 引导层
    MyBatis 提供两种方式来引导MyBatis ：基于XML配置文件的方式和基于Java API 的方式

整体构建结构如下：

每个基于 MyBatis 的应用都是以一个 SqlSessionFactory 的实例为中心，SqlSessionFactoryBuilder 从 XML 配置文件或一个预先定制的 Configuration 的实例构建出 SqlSessionFactory 的实例。
SqlSessionFactory 创建 SqlSession 实例，SqlSession 完全包含面向数据库执行 SQL 命令所需的所有方法。
